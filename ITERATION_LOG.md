# Iteration Log

## v1 — initial sketch

Implemented:

- retained `Node` tree,
- basic `Theme`,
- `Layout::{Column, Row, Split, Overlay, VirtualList}`,
- `RenderCommand`,
- SVG output,
- inspector tree,
- modal motion contact sheet,
- Git-dashboard example.

Weaknesses exposed:

- `desired_w` / `desired_h` encouraged magic pixel constants,
- source maps pointed to component definitions rather than call sites,
- no event/action model,
- no lint artifact,
- no responsive-width artifact.

## v2 — response to Opus review

Implemented:

- `Size::{Fixed, Fill, Hug}` sizing intents,
- `Padding::all` and `Padding::xy`,
- `Align::{Start, Center, End, Stretch}`,
- generic `Node<Msg>` / `El<Msg>` with `.on_action(Msg)` for typed actions,
- tiny `src_here!("Component")` macro for call-site source capture,
- token-carrying `Color` / `TokF32`,
- `lint_tree()` with duplicate ID, token usage, raw color, and raw number reporting,
- `responsive_tape()` artifact rendering the same fixture at 600/900/1200/1600 widths,
- example rewritten to use `Fill`, `Hug`, and typed `Msg` actions.

Artifacts now generated by `examples/git_dashboard.rs`:

- `out/git_dashboard.svg`,
- `out/git_dashboard.inspect.txt`,
- `out/git_dashboard.lint.txt`,
- `out/git_dashboard.motion.svg`,
- `out/git_dashboard.responsive.svg`.

Useful finding from v2:

- The responsive tape immediately reveals the current 3-pane shell is not viable at 600px: the fixed sidebar + fixed inspector consume more than the available width, collapsing the main column. This is exactly the kind of artifact an LLM can use to diagnose responsive failures.

Known remaining problems:

- `src_here!` helps but is not yet an ergonomic component macro; component-created internal title nodes still point to component definitions.
- IDs are still path-derived unless manually set; explicit list keys are recorded but not yet used in ID generation.
- Event handling is only metadata (`action: Option<Msg>`), not a runtime update loop.
- Layout still lacks wrapping, breakpoints, real intrinsic text measurement, and responsive conditionals.
- Token lint currently reports raw colors used intentionally for transparent fills and high-contrast text; these should become named tokens.
- No hit-testing, focus order, interaction storyboard, or real animation runtime yet.

Next likely slice:

1. Make IDs/key paths deterministic and collision-resistant.
2. Add a real `view(state) -> El<Msg>` / `update(msg, state)` example harness.
3. Add responsive primitives: hide/collapse/reflow at named breakpoints.
4. Add one non-trivial component, probably `CommandPalette`.
